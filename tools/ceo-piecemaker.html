      Changelog


      - Visuals: Cost number enlarged and no longer uses monospace font.
      - Visuals: Fixed the strange svg setting that made the outlines of the board look like a sudoku puzzle.
      - Visuals: Letters are more naturally spaced apart.
      - Feature: Can double-click the origin to add an action into the infopanel but not the actions (like RoyalGuard that shows an augmented Teleport) (not actually a new feature).
      - Feature: Can put [+] or 3[+1], for example (including the brackets) to signify unit names that are upgraded or numbers that increase or decrease by the same value across upgrades. "Promotes to Knight[+]. On Death: Lose 4[+1] morale." is usable as the passive for the entire Militia (not actually a new feature).
      - Feature: Can drag from the origin to any adjacent square to create Rook/Bishop lines (reimplemented).
      - Bugfix: Imports can no longer change your selected action for no reason.
      - Bugfix: Empty passives no longer incorrectly display the - in the infopanel (would happen if you unfocused, refocused, then deleted the passive).
      - Bugfix: Right clicking twice on the board no longer puts the cursor in an unfunctional mode that forces you to double left-click to fix it.
      - Bugfix: Importing no longer has strange interactions with empty passives on higher tiers.
      - Feature: You can edit action text straight from in the infopanel and it automatically updates the text. This also works on non-custom abilities but won't save on export. You can also change the names of upgraded pieces, but this doesn't save on export either.
      - Feature: Scrollbar in the sidebar, and also if the screen is too zoomed-in, it will left-align the boards.
      - Feature: Ability to flag SSP's post.
      - Feature: Ability to put things like "v0.50 BonePile" in CEO pretend generator to get old versions of uncollectible units.
      - Feature: You can now use a modified version of Betza notation in the import code, you can export it, and it's generally great. The only thing it doesn't support is custom abilities.
      - Feature: A detailed changelog with my summaries on the meta.
      - Change: The order of the "(Ranged) Destroy target." ability and "(Magic) Teleport unit to Ability Target." have been switched to match the game's order. This breaks a very small amount of previous piecemaker export codes.
      - Change: Action descriptions and icons match v0.54.
      - Historical Inaccuracy: The change involving Portal and Antimage's descriptions, changing to immunity keywords instead of "Immune to all ranged and magic attacks.", isn't known when it happened, between v0.39 - v0.40.
      - Historical Inaccuracy: In the real v0.41, Phoenix++'s infopanel actually showed a "Move or Attack" movetype when it didn't have it, because Grand is a good consistent designer who codes in the displayed movetypes differently than the movetypes the unit actually has. This has been fixed but is historically inaccurate.
      - Historical Inaccuracy: In the real v0.47, GravityMage+ infopanel didn't show the "Move, Attack, or swap places with ally" movetype even though it had it.
      - Historical Inaccuracy: In the real v0.47, v0.48, and v0.49, GravityMage++ infopanel actually showed a "Move or Attack" movetype when it didn't have it.

      - Also, to add a passive in the CD and FC styles, click the space between the board and the actions list.


      Remaining Issues


      This section is to let you know that I'm not doing these on purpose, and I am not that stupid.
      - CEO Unitgen: You can have more than one ability target on the same unit.
      - CEO Unitgen: Due to poor coding choices, it may be possible to get overlapping actions which cause temporary bugs if you try to interact with the overlapping actions. The CEO unitgen code is a lot less polished than the FC unitgen code but at this point I am not rewriting it.
      - CEO Unitgen: There's no actual cost formula.
      - CEO Unitgen: There's no random chance to create a custom ability, like "[Pay 5]: (Magic) Create a RANDOMUNIT, or transform enemy into ally RANDOMUNIT."
      - FC Unitgen: Tokens are kind of a meme. Maybe minions could be more realistic.
      - FC Unitgen: Stuff like "*(Armor Removed)" or "*(Already created a Knight)*".
      - Visuals: In the FC style, an empty actionlist still shows a thin black rectangle.
      - Visuals: The gallery uses terms like "v41" when it should be "v0.41". Too bad, the data is stored as an object and it doesn't like periods in the name. Fixing this would be half a day's wasted time.
      - Visuals: The "fill color" should really be contextual on if you've selected Shape. It's just a bunch of useless settings without Shape selected.
      - Visuals: The size of the infopanel should maybe be the same on all tiers.
      - Visuals: Fancy temporary glow to modified elements (mostly the cost) when you change units in the gallery. This requires some CSS animation nonsense and probably the ability to detect changes between objects.
      - Bug: Some parts of the export may not be saved and maybe the image function will randomly break. I don't know because I haven't used it often.
      - Bug: The autogenerated changelog is summarized. You know, the thing that makes it say "Dragon cost 16/19/22/24 -> 15/18/21/24" or whatever. These summaries probably have annoying bugs. There's one where only the base tier description is used as the example, but the number check removes all the numbers causing this bug - essentially you could have a number that changes across tiers that isn't reflected correctly in the summary. Sometimes it reports a passive added like " Immune to Freeze" with a space (actually a linebreak) and nothing seems to get rid of it despite a "megatrim" function.
      - Feature?: All previous versions for the unit gallery, syncing CEO art, and a search function.
      - Feature?: Double-clicking a non-custom ability should set the next unused custom ability to a clone of whatever you clicked.
      - Feature?: Unit gallery somehow detects reused unit names and passives so that promotes also use the [+] and [+1] convention.
      - Feature?: UI rework, with many buttons being placed on the bottom instead, which is more convenient to scroll down a small amount rather than click some buttons and mess with tabs.
      - Feature?: The custom action editor is only like 8 variables, it doesn't need a full-screen popup! Also a better color selector, or a way to only show one half of a unicode symbol.
      - Feature?: The FC style could maybe have a token style, and the CD style could have a different selector for the "[Minion]-[Basic]-[Rare]".
      - Feature?: In the changelog, clicking names really should automatically make you select the unit.
      - Complaint: A lot of the pretend abilities are bad game design, with probably a good quarter of them literally being copies of other abilities but ignoring status effects, or having slightly different interactions with 1/100 of the game's units. This will never be fixed.
